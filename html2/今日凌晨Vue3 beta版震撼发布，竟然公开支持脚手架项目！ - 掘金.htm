今日凌晨Vue3 beta版震撼发布，竟然公开支持脚手架项目！

尤雨溪微博今日凌晨发布的一则消息打破清晨的宁静！

就在我今早起床准备上班时，突然肚子不太舒服要上厕所，大家也知道上厕所的时候必须得看点什么对吧，于是乎我打开了微博，居然看到了一个熟悉的头像👇

是他，是他，就是他，我们滴大神~小小尤！这则振奋人心的消息令原本平静的清晨变得不再平静。

据报道，北京时间凌晨4点56分，尤雨溪还在为Vue 3.0日夜奋战。

你见过凌晨四点的洛杉矶吗？——Evan You

不对！这是北京时间，尤雨溪身在美国，有时差！美国时间那时候应该还是下午，而且尤雨溪好像也不住洛杉矶，哎呀不管了，就当是尤雨溪说的吧！

让我们点进项目去看一眼↓

来给大家翻译一下：

  • 所有计划好的RFC都已合并
  • 所有合并后的RFC都已实现
  • 现在通过vue-cli-plugin-vue-next插件实验性的支持了Vue CLI脚手架
  • 点击这里查看简易版支持单文件组件的webpack插件

请注意：现在没有写入文档的功能表现依然和Vue 2.x不一致。如果当你运行小案例的时候发现表现和已存在的RFC不一致，请确保初次检查。如果不是一部分的RFC表现不一致，那么可能还没替代，这种情况下可以开个issue给我们（当开新issue的时候请确保使用issue helper）

另外，当前需要本地支持ES2015+的环境才能够运行，并且目前暂不支持IE11。但是以后我们会出一个IE11的兼容版本。

率先尝鲜

既然已经可以通过插件来在vue脚手架上面运行Vue 3 Beta项目了，那就一起来试试吧！

首先，安装@vue/cli脚手架

如果你已经安装过了可以跳过这一步 打开命令行，输入：

npm i -g @vue/cli
复制代码

然后回车

安装过的话再次安装就是升级最新版

安装成功后创建一个项目

(有基础的可以跳过这些照顾新手的步骤往下看)

vue create test
复制代码

按下回车即可安装一个基础版项目。

安装vue-next插件

vue add vue-next
复制代码

回车后会显示如下界面↓

翻译：当前仓库有未提交的文件，推荐进行提交

依然继续吗？(是/否)

这时候输入y然后回车↓

这时候就已经开始大功告成啦！大家赶快按步骤去抢先体验Vue3吧！
